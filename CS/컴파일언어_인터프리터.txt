 # 컴파일 언어와 인터프리터 언어의 차이.

컴퓨터와 대화 하는 방법.
ALU Arithmatic Logical Unit : 산술연산장치.
CU Control Unit : 컴퓨터 시스템을 제어하는 장치.
3+4 -> Compile -> 기계어 -> 실행 -> RAM -> CU -> ALU -> 연산 

컴파일 언어로 만든 프로그램을 실행하는 방법.
한번에 미리 번역한다.
배포시 구문 오류가 존재할 수 없다.
기계어를 직접 만들어 낸다.
실행만 하면 되기 때문에 수행성능이 좋다.
별도의 실행환경이 존재하지 않는다.
메모리 관리도 직접 해야 한다.

인터프리터 언어로 만든 프로그램을 실행하는 방법.
컴파일이 없다.
VM Virtual Machine이 필요하다. ( 실행을 시켜주는 기계 )
수행능력이 조금 떨어진다. ( 기계어로 바꾸면서 바로 실행하기 때문에 )
실행환경에 영향을 받지 않는다.
소스코드가 모두 노출된다.



 # 컴퓨터는 왜 이진수만 이해하고 그것을 이해하는 주체는 누구인가 ?

CPU.
Core안에 ALU와 CU가 존재한다.

어셈블리어.
어떠한 기계에 내장된 기능.
기계어로 변형해줘야 실행가능.
문자코드, 의사코드.

기계어.
어셈블리어의 2진 숫자코드.
직접 실행가능.
숫자코드, 실행코드.
