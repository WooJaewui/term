


 # 추상 메소드와 인터페이스 차이.



상태.

추상 클래스.
● 상태를 가질 수 있으며 해당 메서드는 구현의 상태에 엑세스할 수 있다.

인터페이스.
● 기본 메서드가 허용 되지만 구현 상태에 액세스할 수는 없다.
● 기본 메서드에 작성하는 모든 로직은 다른 메소드와 관련되어야 한다. ( ??? )
	=> 다른 메소드를 사용하지 않으면 상태( 인스턴스 변수 )를 사용하지 않는 것과 같다.
	=> 상태를 사용하지 않으꺼면 default가 아닌 static으로 만드는 것이 좋다 ??



생성자.

추상 클래스.
● 생성자를 가질 수 있으므로 생성시 상태를 초기화할 수 있다.

인터페이스.
● 생성자가 없다. ( 초기화 불가능 )



구문상의 차이점.

추상 클래스.
● Object 클래스 메서드를 재정의할 수 있다.
● 모든 접근 제어자와 함께 인스턴스 변수를 선언할 수 있고, 자식 클래스에서 접근할 수 있다.

인터페이스.
● Object 클래스 메서드를 재정의할 수 없다.
● 인스턴스 변수를 가질 수 없다. ( static final 변수만 사용 )



이해 못한 부분.

● 인터페이스는 둘 중 하나를 가질 수 없다 ( ?? )
● 추상 클래스는 람다 식을 참조할 수 없지만 인터페이스는 람다 식을 참조할 수 있는 단일 추상 메서드를 가질 수 있습니다.
