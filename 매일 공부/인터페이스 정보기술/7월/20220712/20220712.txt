 # 기본적인 네트워크.

이더넷.
- 네트워킹의 한 방식이다. ( 네트워크를 만드는 방법 중 하나 ) 
- 장점.
	- 적은 용량의 데이터를 전송할 경우 성능이 우수하다.
	- 설치 비용이 저렴하고 관리가 쉽다.
	- 네트워크 구조가 단순한다.
- 단점.
	- 네트워크 사용 시에 신호 때문에 충돌이 발생한다.
	- 충돌이 발생하면 네트워크에서 지연이 발생한다.
	- 시스템의 부하가 증가하면 충돌도 계속적으로 증가한다.

CSMA/CD.
- LAN의 통신 프로토콜의 종류 중 하나이며, 이더넷 환경에서 사용한다.
- Carrier Sense Multiple Access/Collision Detection의 줄임말로, "대충 알아서 눈치껏 통신하자" 라는 방식이다.

CSMA/CD 방식.
1. 이더넷 환경에서 통신을 하고 싶은 PC나 서버는 먼저 네트워크 상에 통신이 일어나는지 확인을 한다. ( 캐리어가 있는지 검사 )
2. 네트워크 통신이 일어나고 있으면 ( 캐리어가 감지되면 ) 데이터를 보내지 않고 기다린다.
3. 네트워크 통신이 일어나고 있지 않으면 ( 캐리어가 감지되지 않으면 ) 데이터를 네트워크 상에 보낸다.
4. 캐리어가 감지되지 않았을 때, 두 PC나 서버가 데이터를 동시에 보내면 이 경우를 다중접근 ( Multiple Access )이라 한다.
5. 두 PC나 서버가 데이터를 동시에 보내려다 부딪치는 경우 충돌( Collision )이 발생했다고 한다.
6. 충돌이 일어나게 되면 데이터를 전송했던 두 PC나 서버들은 랜덤한 시간 동안 기다린 다음 다시 데이터를 전송한다.
7. 충돌이 계속해서 15번 일어나면 통신을 끊는다.

캐리어.
- 네트워크 상에 나타나는 신호.

 # SDN

SDN Software-Defined Networking.
- 네트워킹 리소스를 가상화된 시스템으로 추상화하는 IT 인프라에 대한 하나의 접근 방식이다.
- 중앙에서 관리하고 프로그래밍할 수 있는 네트워크를 구축하기 위해 네트워크 포워딩 기능을 네트워크 제어 기능에서 분리.



 # IntelliJ.

단축키.
- Alt + Enter : 경고나 오류가 있을 때 수정 방법을 제시해준다.
- F2 : 다음 경고나 오류로 점프뛰어 준다.
- Alt 1 ~ 9 : 창들을 열고 닫을 수 있다.
- ESC : 포커스를 에디터 창으로 가져올 수 있다.
- ctrl + E : 최근에 열었던 파일을 확인할 수 있다.
- ctrl + b : 코드의 선언부분으로 이동할 수 있다.
- Alt + F7 : 메서드에서 사용하면 메서드가 어디서 호출되는지 모두 확인할 수 있다.
- ctrl + ctrl : RUN 창이 뜨고 어떤 항목이든 실행할 수 있다.
- ctrl + shift + enter : 현재 구문 완성.

팁.
- 파일 우클릭/다이어그램을 선택하면 자바 또는 스프링 계층관계 다이어그램을 볼 수 있다.
