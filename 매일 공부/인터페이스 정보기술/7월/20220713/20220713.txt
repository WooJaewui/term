 # Monolithic Kernel vs Micro Kernel.

Monolithic Kernel.
- Application을 제외한 모든 system 관련 기능들( VFS, IPC, Filesystem 등 )을 커널이 관리하며 각 영역들은 단계적으로 나뉘어 있다.
- Unix, Lunux 등.

Monolithic Kernel 장단점.
- 장점.
	- 각 Componenet간의 커뮤니케이션이 효율적이다.
- 단점.
	- 디바이스 드라이버를 추가/삭제 하려면 커널을 재빌드 해야 한다.
	- 하나의 시스템이 죽으면 전체 시스템이 죽는다.

Micro Kernel.
- 핵심적인 기술( 스케쥴링, 메모리 관리 등등 )만 커널에 담고 나머지는 제외해 가볍게 만든 커널.
- 기존에 모놀리식 커널이 갖고 있던 시스템 기능들( VFS, IPC, Device Driver )은 커널위의 서버의 형태로 존재한다.
- 하나의 서비스가 죽더라도 커널 전체가 죽는 일이 없다.
- MacOs X, Windows NT 등.

Micro Kernel 장단점.
- 장점.
	- 서버를 추가하는 방식이기 때문에 기능을 추가하기 쉽다.
	- 시스템이 견고하며 리얼타임성이 높다.
- 단점.
	- 시스템 기능들이 서버의 형태로 존재하기 때문에 커뮤니케이션 오버헤드가 있다.

모듈형 커널.
- Micro Kernel과 Monolithic Kernel을 합쳐놓은 형태의 커널.
- 모듈의 형태로 시스템 기능을 추가 할 수 있는 방식이다.
- 시스템을 재부팅할 필요 없이 커널의 기능을 확장한다.
- 윈도우 등.

LKM Loadable Kernel Module
- 커널 소스트리의 재빌드 없이 Linux 커널이 실행되는 동안 코드를 추가할 수 있다. ( Monolithic Kernel의 단점을 보완 )
- 모듈이 커널의 한 부분처럼 동작한다.
- 디바이스 드라이버는 모듈 형태로 구성되어 있다.


 
 # intellij

svn depth.
- Working copy : 아직 체크 아웃되지 않은 저장소 하위 트리에서 파일/디렉토리를 가져오려면 이 옵션을 선택합니다.
- Empty: 현재 파일만 포함하려면 이 옵션을 선택합니다.
- Files: 폴더의 파일을 포함하려면 이 옵션을 선택합니다.
- Immediates: 현재 파일의 직계 자식을 포함하려면 이 옵션을 선택합니다.
- Infinity: 전체 재귀를 활성화하려면 이 옵션을 선택합니다.

.idea 폴더.
- 프로젝트 별 설정을 저장하는 폴더.
- 버전 관리에서 ignore해야 될 폴더이다.
